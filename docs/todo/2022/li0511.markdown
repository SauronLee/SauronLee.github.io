研究２：
（１）発見：
	１）論文の文体は、単純な文法と専門的な語彙を持っているべく。つまり、論文スタイルは単純な文法で複雑な意味を表現する。
	２）文法の複雑さを表す方法を探した。
		- 依存距離（Dependency Distance）「1」（この方法を使っている人が少ない）
（２）考え：
	１）タスクの評価はいつも困難な問題でした、手作業で作る評価データは言語学者あるいは権威のある機関・団体が行っている。
	　　自分の考え、評価データが要らなくで評価モデルを作る：
		１）論文スタイルを評価できる分類器を作成する。この分類器は普通の文章と論文を分類することができる。
		２）文章の流暢さを評価できる言語モデルを作成する。この言語モデルは文章の流暢さを評価できる。

（３）試み：
	１）依存距離が分類器の収束を加速させるかどうか。
		依存距離が小さいほど、文章の文法は単純になります。以下の試験は依存距離が２、３、４、５で設定して同じのTestデータセットで行われていた。
			- モデル：CNN {embedding:300, batch:128, dropout:0.5, pad_size:32, filter_size:(2,3,4)}
				2：Test Loss: 0.083	Test ACC 96.91%		Time: 	    0:40:07	Iter: 1300
				3：Test Loss: 0.053	Test ACC 97.80%		Time: 	    3:17:29	Iter: 6500
				4：Test Loss: 0.055	Test ACC 97.79%		Time: 1 day 6:42:30	Iter: 5600
				5：Test Loss: 0.055	Test ACC 97.75%		Time: 1 day 9:59:15	Iter: 6300
			結論：効果は明ら​​かではありません、この実験は無駄に行われました。

	２）論文スタイルを評価できる分類器を作成した。
		- モデル：CNN {embedding:300, batch:128, dropout:0.5, pad_size:32, filter_size:(2,3,4)}
			Test Loss: 0.054	Test ACC 97.92%	(最優)	Time: 1 day 15:46:55	Iter: 8000

	３）文章の流暢さを評価できる言語モデルを作成した。
		- モデル：
			Kenlm N-gram {window_size:3}
			Kenlm N-gram {window_size:4}
			Kenlm N-gram {window_size:5}
